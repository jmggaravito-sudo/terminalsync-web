## Cuándo usarlo

- Querés una auditoría SEO de tu sitio **sin** tener que copiar y pegar el HTML de cada página a mano.
- Tenés una URL pública y querés los problemas ordenados por impacto, con evidencia y un veredicto claro de por dónde empezar.
- Sos dueño de negocio o marketer y querés una primera revisión honesta —no un checklist genérico ni promesas de ranking.

## Qué hace

Junta dos piezas que se potencian, en un solo install:

- **Firecrawl (el conector)** baja el contenido limpio de tus páginas para que el análisis trabaje sobre lo que tu sitio realmente muestra —incluyendo lo que se renderiza, no solo el HTML crudo.
- **SEO Auditor (la skill)** analiza eso y te devuelve los bloqueantes primero, los fixes de alto impacto, la evidencia de cada hallazgo y un **veredicto puntuado** (🟢/🟡/🔴) con el único arreglo de mayor impacto para arrancar —**sin garantías de ranking**.

**Un ejemplo real:** tenés una landing de "pricing" que no convierte. Le decís *"auditá el SEO de misitio.com/pricing"*. Firecrawl baja la página, el SEO Auditor detecta que el `title` no coincide con el `H1`, que falta el schema de producto y que la meta description está duplicada con la home, y te da un veredicto 🟡 62/100 con "arreglá primero el title/H1" como próximo paso. Vos sabés exactamente qué tocar.

## Cómo usarlo

1. Instalá el Plugin y conectá **Firecrawl** con su API key.
2. Pedí: *"auditá el SEO de mi sitio https://misitio.com"* (o una página puntual).
3. Firecrawl baja las páginas; el SEO Auditor las revisa.
4. Recibís los fixes priorizados, la evidencia y el veredicto —listos para actuar.

## Por qué el combo funciona

Instalados por separado, tendrías que saber que el SEO Auditor **necesita** ver tus páginas para trabajar bien, e ir a buscar una herramienta que se las baje. El Plugin resuelve eso: la skill que analiza y la herramienta que consigue el material vienen juntas y ya conectadas. Es la diferencia entre *"pegame el HTML de cada página"* y *"auditá mi sitio"*.

## Límites

- Solo puede auditar lo que Firecrawl pueda acceder: páginas públicas sí; áreas con login, analytics privados o Search Console requieren acceso aparte.
- No garantiza rankings, tráfico ni ingresos —el veredicto es la lectura del modelo sobre lo que vio, no una promesa. Los cambios se validan midiendo después.
- Firecrawl necesita su API key y está sujeto a los límites de tu plan.
